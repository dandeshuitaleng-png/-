# 初赛作品说明：今天吃什么 2.0

## 作品基本信息

| 项目 | 内容 |
| --- | --- |
| 作品名称 | 今天吃什么 2.0 |
| 作品类型 | iOS 饮食决策助手 |
| 目标平台 | iOS，后续可扩展 iPadOS / visionOS |
| 目标用户 | 高校学生、办公园区年轻人、午晚餐选择困难用户 |
| 核心价值 | 在预算、忌口、距离、出餐速度和健康偏好约束下，帮助用户快速做出一餐选择 |

## 一、问题背景与用户分析

高校学生和年轻上班族每天都会面对“吃什么”的小决策。传统外卖和点评平台提供大量商家、折扣和评价，但默认用户已经知道想吃什么；当用户处于饥饿、赶课、午休时间短或多人意见不一致的场景时，过多选项反而会增加决策成本。今天吃什么 面向这类“需要快速决定，而不是继续浏览”的用户，将一餐选择拆成预算、距离、忌口、出餐速度和健康目标几个关键因素，用更少操作给出可执行推荐。

## 二、相关竞品分析

美团、饿了么和大众点评的优势是商家覆盖、交易链路和评价体系，适合“我想吃火锅，找一家店”的搜索场景；小红书适合发现新店和内容种草，但决策链路更长。今天吃什么 2.0 的差异化不是替代交易平台，而是放在决策前一步：当用户没有明确目标时，先用两题问答和偏好筛选缩小范围，再给出一个主推荐和少量备选。产品强调“少看、快选、少后悔”，避免把用户再次推回无限列表。

## 三、可行性分析

2.0 可以分阶段实现。初赛阶段采用 iOS 高保真原型和 SwiftUI MVP 骨架，验证完整用户路径：打开 App、选择预算和忌口、选择距离和当下状态、直接推荐或回答两题、查看推荐理由、收藏结果。数据层使用 30 条校园周边样本餐饮数据，字段覆盖店名、品类、人均价格、距离、辣度、健康标签、出餐速度和营业时段，用来验证推荐链路是否可行。复赛阶段可接入 CoreLocation 获取用户位置，结合 MapKit 或第三方餐饮数据源展示附近商家，并通过本地历史记录提升个性化权重。

## 四、App 创新点

今天吃什么 2.0 的创新点在于把“随机”升级为“有约束的轻决策”。第一，推荐不是纯随机，而是结合预算、距离、忌口、出餐速度、健康标签和用户当下状态，让结果更可信。第二，产品提供两种入口：时间紧时一键推荐，想更贴合心情时回答两题，兼顾效率和参与感。第三，结果页不只展示菜系或店铺，还解释推荐理由，例如“预算内、可选辣度、步行可达、出餐快”，降低用户对算法的黑箱感。第四，产品保留收藏和历史偏好，为后续个性化推荐、校园餐饮合作和群体决策打基础。

## 五、应用前景

该产品适合从高校和办公园区切入，因为这些场景餐饮选择集中、决策频次高、用户时间敏感。短期可以做成个人工具，帮助用户减少午晚餐纠结；中期可以接入校园周边餐厅、食堂窗口、优惠信息和营养标签；长期可以扩展到多人投票、低碳饮食、预算管理和健康计划。与大平台不同，今天吃什么 2.0 的价值在于更靠近“决定吃什么”的瞬间，适合作为轻量、高频、可持续迭代的移动应用。

## 六、初赛演示路径

1. 首页展示样本数据规模、预算、距离、忌口和状态设置。
2. 点击“一键决定”，系统给出主推荐、推荐理由和 3 个附近备选。
3. 切换“两题模式”，选择“赶时间、清淡、管饱、奖励”等状态后生成更贴合当下的结果。
4. 将满意的推荐加入收藏，下一次推荐会提高同类结果权重。
